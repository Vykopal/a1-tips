c08.1 - Component API - lifecycle-hooks

Lifecycle-hooks simply let you intercept some important events in the life of your component. This model is implemented in Angular2 from the very beginning through interface implementation, and I have a slight suspicion, that it was back-implemented to AngularJS.

On this page we're gonna demonstrate this essential life-cycle hooks:

  • $onInit - called on each controller after all the controllers on an element have been constructed and had their bindings initialized.
  • $postLink - called after this controller's element and its children have been linked.
  • $onChanges - called whenever one-way bindings are updated.
  • $onDestroy - called on a controller when its containing scope is destroyed.
  • ...there is more in AngularJS documentation
