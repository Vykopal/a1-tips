Address directive

Think - do we always need such behavior - I mean inheriting parent scope? Especially in case of directives?

Address directive - isolated scope

The is one of the very best practises when writing directives - let them have tehir scopes isolated, not to mess up with other scope models in the same view.


Conclusion

  • Be aware of scope inheritance
  • Isolate scopes for directives, if you need let them accept the parent model explicitly
  • Try to write your directives in a component directive manner, or use component API (AngularJS ver. >= 1.5)
