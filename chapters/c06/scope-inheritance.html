c06 - Scope inheritance

Controller - the parent scope (ScopeInheritanceCtrlParent)

$scope.customer = 'Abraham';
$scope.address = {
    streetName: 'Husakova',
    streetNumber: '1971'
}

scope: false

Two-way, full prototype inheritance

scope: true

Automatically done implicit input binding.

scope: {}

Isolated scope, no bindings

scope: {customer: '='}

Explicitly stated 2-way binding

scope: {customer: '<'}

Explicit input-only binding

Due to documentation, you can use scope = {'@'} as well, especially for strings that do not change often.

Conclusion

  • Be aware of scope inheritance - these little things create deep mess in the final result!
  • Build your application from components and directives, not from controllers (A2 way:-))
  • Isolate scopes for directives, only let them accept parent model explicitly
  • Try to write your directives in a component directive manner, or use component API (AngularJS ver. >= 1.5)
